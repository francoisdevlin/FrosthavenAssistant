# Wishlist — Game Night Irritations

Things that bug us when we play. Captured fresh, no filter. Triaged later.

## Format

For each entry:
- **What:** the irritation, in one or two sentences
- **When:** which moment of play it happens
- **Severity:** trivial / annoying / blocking
- **Upstream issue:** link if one exists, blank if not
- **Notes:** thoughts, possible approaches, related ideas

Don't pre-filter. If it bugs you, write it down. Triage and de-duplication happen later, not at capture time.

## Triage backlog → strategy

When we have a few entries:
1. Cross-reference against upstream's open issues. Many irritations are already filed.
2. Pick the one with highest (annoyance × frequency) and lowest implementation cost as the next Tier-2 candidate.
3. The ones that are too opinionated for upstream become fork-only branches off `fork-main`.
