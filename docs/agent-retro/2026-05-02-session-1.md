## What I had to correct

- **`bd init --stealth` modified `.gitignore`** in addition to `.git/info/exclude`. Caught when switching to a PR branch — the leaked entries would have polluted upstream PRs. Lesson: stealth mode in this version of `bd` is *partial* stealth. Always re-check `git status` after `bd init`.
- **Premature `-c commit.gpgsign=false`.** Adding signing-bypass flags preemptively. Stopped, owned it, removed from future commits. Good rule: never add `--no-verify`, `--no-gpg-sign`, etc. unless explicitly asked.
- **Underestimated Flutter version churn.** First tried 3.27.2 (matches CI but too old for current deps). Then 3.32.8 (still too old). Then 3.35.7 (right Dart version but hit pub.dev `advisoriesUpdated` decode bug). Lesson: project's `pubspec.lock` had drifted past upstream's CI pin — the CI itself is broken on `main`. Worth flagging upstream eventually as a separate small fix.
- **Over-eager rulebook ambition.** Initially stubbed `docs/games/{gloomhaven,frosthaven,...}` for rulebook material before the user pointed out the cleaner sibling-private-repo pattern. Should have proposed the structure question first, not committed to a structure.

## What would have made this faster

- **Knowing about `claude_refactor` from the start.** Tier-0 should always include checking ALL non-default branches on the upstream, not just merged PRs.
- **Explicit `bd init` documentation around stealth-mode side effects.** Cost ~10 minutes of confusion when the leaked `.gitignore` showed up on a PR branch.
- **A `flutter` "doctor for project" command** (something like `flutter pub get --dry-run`) that surfaces SDK version requirements without trying to download. Would have saved us from downloading and switching versions twice.

## Carry-forward

These should become standing checks for future sessions:

1. Tier-0 lurking includes ALL upstream branches, not just merged PRs.
2. After `bd init --stealth`, `git status` to confirm no `.gitignore` modification.
3. Disclose AI assistance on every public-facing comment / PR.
4. Read `claude_refactor` for context before any non-trivial UI/Layout work.
