# X-Haven Assistant — User Manual

A section-numbered HTML manual for the app. Anchored to `upstream/main`; the
PNG goldens render byte-identically on the `claude_refactor` branch
(verified with side-by-side worktrees), so the same screenshots can serve
as a regression-test artifact for verifying the refactor preserves UI.

Open [`index.html`](index.html) in a browser to read.

## Layout

```
docs/
  manual/
    index.html                    TOC + status banner
    01-welcome.html
    02-quick-start.html           Black Barrow walkthrough (anchor scenario)
    03-anatomy.html               Top bar, main list, bottom bar, drawer
    04-setting-up.html
    05-round-step-by-step.html
    06-special-mechanics.html
    07-frosthaven.html            Loot, sanctuary, Buttons & Bugs
    08-edition-differences.html
    09-multiplayer.html
    10-mistakes-undo.html
    11-settings.html              Single grouped table
    12-walkthrough.html           Long-form Black Barrow narrative
    13-reference.html             Menu map, gestures, icons, shortcuts
    shared.css                    Single stylesheet linked from every chapter
    README.md                     This file
  screenshots/manual/
    sN-M-*.png                    Goldens, named after the manual section
                                  they illustrate
```

Section numbers (`§N.M`) are stable identifiers — tests, bug reports, and
cross-references can cite them. Each chapter file owns one top-level
`<section id="sN">` and its `<h3 id="sN-M">` (or `<th id="sN-M">`) sub-sections.

## How to regenerate

Three regenerators, one per artifact kind. Run them in this order when the
app changes:

### 1. Screenshots (PNG goldens)

Generated by Flutter widget tests at
[`frosthaven_assistant/test/widget/manual_goldens_test.dart`](../../frosthaven_assistant/test/widget/manual_goldens_test.dart).

```bash
cd frosthaven_assistant
flutter test test/widget/manual_goldens_test.dart                    # verify
flutter test --update-goldens test/widget/manual_goldens_test.dart   # rewrite
```

The test file declares one `testWidgets('sN-M ...')` block per golden, named
after the manual section that embeds it. Helpers in the same file:

| Helper | Purpose |
|---|---|
| `_setUpGameWithFullData()` | Replaces `setUpGame()` to load production data (`assets/data/`) instead of testData, so the Gloomhaven edition is reachable for Black Barrow content. |
| `_setupBlackBarrow({withSpellweaver})` | Adds the manual's anchor roster (Brute, optional Spellweaver, Bandit Guard with normal+elite, Bandit Archer normal). |
| `_pumpFullApp(tester)` | Pumps the full app shell (TopBar + BottomBar + drawer + MainList) for full-screen captures. |
| `_pumpInScaffold(tester, body, ...)` | Pumps an arbitrary widget in a themed Scaffold for component-level captures. |
| `_loadAppFonts()` | Registers Pirata, GermaniaOne, Markazi, Majalla, and MaterialIcons. Without this, glyphs render as boxes. |
| `_precacheAllImages(tester)` | Forces `Image` and `DecoratedBox`-with-image-decoration to decode before capture. |

To add a new screenshot: add a `testWidgets('sN-M short name', ...)` block
that pumps the desired widget and calls `expectLater(..., matchesGoldenFile(...))`,
re-run with `--update-goldens`, then `<img src="../screenshots/manual/sN-M-*.png">`
in the chapter file.

### 2. Sidebar and sub-TOC

Generated by [`scripts/inject_nav.py`](../../scripts/inject_nav.py). Idempotent
— re-running on unchanged input produces byte-identical output.

```bash
python3 scripts/inject_nav.py
```

For each chapter file, the script:
1. Extracts sub-sections by finding `<h3 id="sN-M">` and `<th id="sN-M">` elements.
2. Regenerates the **left sidebar** listing all 13 chapters; the current
   chapter is marked `.active` and its sub-sections expanded.
3. Regenerates the **"In this chapter" sub-TOC** at the top of the content area.

Generated regions are bounded by HTML comments:

```html
<!-- BEGIN GENERATED SIDEBAR -->
...
<!-- END GENERATED SIDEBAR -->

<!-- BEGIN GENERATED SUBTOC -->
...
<!-- END GENERATED SUBTOC -->
```

**Don't hand-edit inside those blocks** — they get clobbered on the next run.

Run inject_nav.py whenever you add, remove, or rename a sub-section heading.

### 3. Figure numbering

Generated by [`scripts/number_figures.py`](../../scripts/number_figures.py).
Idempotent.

```bash
python3 scripts/number_figures.py
```

For each chapter file, finds every `<figure>` element in document order and:
1. Sets `id="fig-N-M"` on the `<figure>` (where N is the chapter number and
   M is the chapter-scoped sequential index — counts continue across
   subsections, restart at chapter boundaries).
2. Prefixes the `<figcaption>` with `<strong>Figure N.M.</strong>` so
   readers and external references can cite "§4.2, Figure 4.1" precisely.

Run after adding, removing, or reordering any `<figure>` block. Cross-section
deep links are stable: `<a href="07-frosthaven.html#fig-7-2">Figure 7.2</a>`.

### 4. Chapter prose

Hand-edit `docs/manual/0N-slug.html` directly. Each chapter is self-contained:
its own `<title>`, link to `shared.css`, prev/next/contents nav, body content,
nav again at the bottom. The split is mechanical — `<section>` boundaries are
preserved from the original single-doc draft via
[`scripts/split_manual.py`](../../scripts/split_manual.py) (kept as a record
of how the migration was done; not rerun).

Cross-references between chapters use `0N-slug.html#sN-M`, e.g.
`<a href="04-setting-up.html#s4-1">§4.1 Add Character</a>`. Same-chapter links
stay as `#sN-M`.

## End-to-end "the app changed" flow

```bash
cd ~/.../FrosthavenAssistant      # working clone
git pull origin main              # latest app
git checkout manual-draft-main    # manual branch

# See what diverges
cd frosthaven_assistant
flutter test test/widget/manual_goldens_test.dart

# Bless new goldens (review the diffs first!)
flutter test --update-goldens test/widget/manual_goldens_test.dart

cd ..
# If you also added/removed sub-sections, regenerate nav
python3 scripts/inject_nav.py

# If you added, removed, or reordered figures, renumber them
python3 scripts/number_figures.py

# Update prose if behavior changed
$EDITOR docs/manual/0N-*.html

git add docs/screenshots/manual/ docs/manual/ frosthaven_assistant/test/widget/manual_goldens_test.dart
git commit -m "docs: refresh manual for <change>"
```

## Cross-branch portability

The PNG goldens are **byte-identical** when generated on either
`upstream/main` or `upstream/claude_refactor` (verified with side-by-side
worktrees). The refactor extracted ViewModels but did not change rendered
widget output for the regions the manual screenshots — so one set of PNGs
serves both branches.

The test file targets `upstream/main`'s positional Add* command API. To run
the same suite on `claude_refactor` (which made `gameState:` a required
named arg), apply this conversion to the test file:

```bash
sed -i 's/, gameState: gs)/)/g' frosthaven_assistant/test/widget/manual_goldens_test.dart
# (revert with git checkout when done)
```

A small upstream PR making `gameState:` optional with a default to
`getIt<GameState>()` on the refactor branch would let one unmodified test
file work on both branches.

## Conventions

- **Anchor scenario:** Black Barrow (Gloomhaven Scenario 1) with a Brute and
  optionally a Spellweaver. Most goldens use this roster. §4.4 (Add Section)
  switches to Gloomhaven 2E #19 *Military Outpost* because Black Barrow
  carries no sections.
- **Section numbers don't move.** If a chapter restructures, prefer adding
  new sub-section numbers over renumbering existing ones — external links
  (issues, BGG threads) cite `§X.Y`.
- **Table-only chapters use grouping rows** rather than `<h3>` headings.
  See `11-settings.html` for the pattern: `<tr class="group"><th colspan="N"
  id="sN-M">...</th></tr>` — `inject_nav.py` picks up the `id` either way.
- **Don't duplicate rulebook content.** The manual covers the *app*; game
  rules belong to Cephalofair's rulebook. Cite, don't replicate.
